# fake-ui v3.0.0

v3.0.0 新增公开发布可用的本地服务发布能力。管理员只需要让客户把域名 A 记录解析到 VPS，在面板填写公网域名、后端系统、本地地址和本地端口，fake-ui 会自动生成 VPS 侧 Xray portal、Nginx HTTPS 入口、Let’s Encrypt 证书，以及 macOS / Linux / Windows Xray bridge 安装包。

## 重点

| 类型 | 内容 |
| --- | --- |
| 通用域名 | 不绑定固定域名后缀，任意 `app.example.com` 这类客户域名都可以创建发布服务 |
| 默认单服务单 bridge | 每个公网域名对应一个 portal 端口和一个后端 bridge，隔离清楚，便于排障 |
| 高级共享 Agent | 同一台后端机器上的多个 Web 服务可以共享一个 Xray Agent 进程；SSH 或救援入口仍建议独立 |
| 身份隔离 | 普通代理用户使用 `panel-user:<username>`，服务发布使用 `tunnel:<id>`，UUID、portal tag、reverse tag 和端口均校验唯一 |
| 一键应用 | 面板应用后写入 Xray、Nginx、证书并 reload，不改变公网 443 上 `Reality SNI -> Xray` 的 stream 分流 |
| 多平台配套 | 导出包支持 macOS launchd、Linux systemd 和 Windows Scheduled Task，均包含 Xray 配置、安装、卸载和状态检查脚本 |

## 升级说明

从 v2.3.1 升级无需数据库 schema 迁移。已有代理节点和用户订阅不变。本地服务发布使用新的 `tunnel_catalog` 设置项保存，未创建发布服务时不会改变现有 Xray/Nginx 行为。

如果服务器上存在旧版手工测试留下的 `/etc/nginx/conf.d/fake-ui-tunnel-*.conf` 单域名配置，应用 v3.0.0 服务发布入口时会自动改名禁用，避免旧 upstream 抢占新域名导致 502。
