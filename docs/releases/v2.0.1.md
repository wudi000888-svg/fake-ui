# fake-ui v2.0.1

v2.0.1 是 v2 商业化面板的稳定修复版，重点修复 Hysteria2 管理页、节点出口信息同步、GitHub 展示素材和 CI 测试隔离，并已在新加坡 VPS 完成安装验证。

## 主要变化

| 模块 | 更新 |
|---|---|
| Hysteria2 管理 | 恢复独立 H2 管理页，支持 HTTP/SOCKS5 上游、恢复直连、状态、日志和移动端展示 |
| 节点同步 | 节点保存、刷新、启停和 H2 出口变更后，前端会应用后端返回的节点数据，立即同步出口 IP、国家和节点名称 |
| 错误提示 | 面板 API 网络失败会展示更明确的连接错误，减少只有 `Failed to fetch` 的情况 |
| GitHub 展示 | README 使用最新 v2 桌面和移动端截图，覆盖 Hysteria2 管理和节点出口同步页面 |
| CI 稳定性 | 测试 fixture 隔离 H2 配置、机场用户日志和支付日志路径，避免 GitHub runner 访问 `/root` 权限失败 |
| 新加坡验证 | 已安装到新加坡测试 VPS，验证 Panel、Nginx、Xray Reality、Hysteria2、DNS、静态资源和 API 基础状态 |

## 部署说明

全新 VPS：

```bash
sudo bash scripts/install-fresh-vps.sh
```

已有 Docker Compose 部署建议先备份 `data/` 和 `.env`，再用最新版源码覆盖并重建：

```bash
docker compose build panel xray
docker compose up -d xray panel hysteria2
```

生产环境不要把 `.env`、`data/`、证书私钥、真实 RPC API Key、真实收款地址或任何钱包私钥提交到仓库。
