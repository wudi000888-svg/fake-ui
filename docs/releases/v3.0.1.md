# fake-ui v3.0.1

v3.0.1 给内网穿透后端 Agent 安装包加入本地可视化控制台。客户下载 macOS / Linux / Windows bridge 包后，可以在后端机器上运行随包提供的 `open-dashboard` 脚本，打开 `http://127.0.0.1:19090/` 查看状态。

同一个 v3.0.1 release 也附带通用本机客户端资产。客户端资产不包含任何客户域名、UUID、私钥、portal 或真实 `xray-bridge.json`；真实配置仍由 fake-ui 面板生成并导入客户端。

## 重点

| 类型 | 内容 |
| --- | --- |
| 本地控制台 | 每个专用 Agent 和共享 Agent 包都包含 `bridge-dashboard.py`、`bridge-dashboard.json` 和平台打开脚本 |
| 仅本机访问 | 控制台默认只监听 `127.0.0.1:19090`，不会暴露到公网 |
| 状态可视化 | 页面展示 bridge 类型、后端系统、运行时服务名、Xray 配置文件、服务列表和本地端口连通性 |
| 通用客户端资产 | 同一 release 提供 macOS zip、Linux tar.gz、Windows zip，方便客户先安装客户端再导入面板配置 |
| 兼容原模式 | launchd / systemd / Scheduled Task 后台运行方式不变，不影响 VPS 侧 portal、Nginx HTTPS 或普通代理节点 |

## 客户端资产

- `fake-ui-bridge-client-v3.0.1-macos.zip`
- `fake-ui-bridge-client-v3.0.1-linux.tar.gz`
- `fake-ui-bridge-client-v3.0.1-windows.zip`

## 升级说明

无需数据库 schema 迁移。已有 v3.0.0 穿透节点继续可用；重新从面板下载后端安装包即可获得本地控制台。
