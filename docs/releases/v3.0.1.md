# fake-ui v3.0.1

v3.0.1 给内网穿透后端 Agent 安装包加入配对 bootstrap 和本地可视化控制台。管理员可以在面板生成配对 Agent 包，客户下载 macOS / Linux / Windows 包后运行安装脚本，Agent 会主动向 fake-ui 拉取自己的 Xray bridge 配置并启动本地控制台。

同一个 v3.0.1 release 也附带通用本机客户端资产。客户端资产不包含任何客户域名、UUID、私钥、portal 或真实 `xray-bridge.json`；真实配置可通过配对 profile bootstrap，也可以继续由 fake-ui 面板导出后手动导入。

## 重点

| 类型 | 内容 |
| --- | --- |
| 配对 Agent | 新增 dedicated/shared `*-agent-bundle`，包内包含 `agent-profile.json` 和 `bootstrap-agent.py`，一次性 token 成功使用后会在本地清空 |
| 面板教程 | “内网穿透”页新增客户操作三步教程，先安装后端客户端，再运行本地服务，最后检查 `127.0.0.1:19090` |
| 单 Agent 入口 | shared bridge 会按 Bridge ID 合并展示，一个后端客户端只显示一次“下载配对安装包”，下面可承载多个服务映射 |
| 本地控制台 | 每个专用 Agent 和共享 Agent 包都包含 `bridge-dashboard.py`、`bridge-dashboard.json` 和平台打开脚本 |
| 仅本机访问 | 控制台默认只监听 `127.0.0.1:19090`，不会暴露到公网 |
| 状态可视化 | 页面采用 fake-ui 风格 shell，展示运行状态、服务探测、导入配置、使用说明、日志和 `GET /status.json`，并对日志/配置预览做敏感值脱敏 |
| 手动导入修复 | 本地控制台导入 `xray-bridge.json` 时会同步刷新服务列表，支持从文件名/已有 metadata 保留公网域名，并拒绝把 Xray 配置误导入 dashboard/profile |
| TUN 共存 | 重新导入配置时保留本机已设置的 reverse address 和 `sockopt`，便于 Shadowrocket、Clash 等 TUN 环境把 VPS IP 设为直连网卡 |
| 通用客户端资产 | 同一 release 提供 macOS zip、Linux tar.gz、Windows zip，包含安全的 profile 示例和 bootstrap helper |
| 兼容原模式 | launchd / systemd / Scheduled Task 后台运行方式不变，不影响 VPS 侧 portal、Nginx HTTPS 或普通代理节点 |

## 使用方式

1. 管理员在“内网穿透”页创建服务映射。多个 Web 服务可以使用同一个 shared Bridge ID；SSH、数据库等隔离场景可以使用 dedicated bridge。
2. 在“后端客户端”区域点击“下载配对安装包”。同一个 shared Bridge ID 只需要下载一个包；新增同 Bridge ID 的服务通常不需要客户重新安装。
3. 客户解压包并运行 `install-macos.sh`、`install-linux.sh` 或 `install-windows.ps1`。
4. 安装脚本先运行 `bootstrap-agent.py`，写入本地 `xray-bridge.json`、`bridge-dashboard.json`、`agent-state.json`，然后继续安装 Xray 后台服务。
5. 客户运行 `open-dashboard`，访问 `http://127.0.0.1:19090/` 查看状态；服务映射的编辑、启停和删除仍在面板“服务映射”区域完成。

手动 JSON 导出仍然保留，用于离线环境、客户机器无法访问面板或排障场景。当前版本不包含面板远程执行命令、远程日志流或 Agent 自动更新；这些能力只保留字段和接口命名空间，避免未来扩展时破坏兼容性。

## 升级说明

无需数据库 schema 迁移。已有 v3.0.0 穿透节点继续可用；重新从面板生成配对 Agent 包即可获得 bootstrap 和新版本地控制台，原静态后端安装包和 JSON 导出仍可继续使用。
