# fake-ui v3.0.1

v3.0.1 给内网穿透后端 Agent 安装包加入配对 bootstrap 和本地可视化控制台。管理员可以在面板生成配对 Agent 包，客户下载 macOS / Linux / Windows 包后运行安装脚本，Agent 会主动向 fake-ui 拉取自己的 Xray bridge 配置并启动本地控制台。

同一个 v3.0.1 release 也附带通用本机客户端资产。客户端资产不包含任何客户域名、UUID、私钥、portal 或真实 `xray-bridge.json`；真实配置可通过配对 profile bootstrap，也可以继续由 fake-ui 面板导出后手动导入。

## 重点

| 类型 | 内容 |
| --- | --- |
| 配对 Agent | 新增 dedicated/shared `*-agent-bundle`，包内包含 `agent-profile.json` 和 `bootstrap-agent.py`，一次性 token 成功使用后会在本地清空 |
| 面板教程 | “内网穿透”页新增客户操作三步教程，先安装后端客户端，再运行本地服务，最后检查 `127.0.0.1:19090` |
| 单 Agent 入口 | shared bridge 会按 Bridge ID 合并展示，一个后端客户端只显示一次“下载配对安装包”，下面可承载多个服务映射 |
| 本地控制台 | 每个专用 Agent 和共享 Agent 包都包含 `bridge-dashboard.py`、`bridge-dashboard.json` 和平台打开脚本 |
| 仅本机访问 | 控制台默认只监听 `127.0.0.1:19090`，不会暴露到公网 |
| 状态可视化 | 页面采用 fake-ui 风格 shell，展示运行状态、服务探测、导入配置、使用说明、日志和 `GET /status.json`，并对日志/配置预览做敏感值脱敏 |
| 手动导入修复 | 本地控制台导入 `xray-bridge.json` 时会同步刷新服务列表，支持从文件名/已有 metadata 保留公网域名，并拒绝把 Xray 配置误导入 dashboard/profile |
| 域名资格 | “内网穿透”会从候选/已有域名中自动识别已解析到本 VPS 且未被面板、普通节点或其他穿透占用的域名；也支持输入新域名并在保存时校验 |
| TCP/SSH | 私有 TCP 服务明确标注“无需域名”；SSH 作为全局默认私有 TCP 服务，不再按每个域名或每个 Bridge 反复生成，也不会在面板或本地控制台里显示具体公网域名 |
| 总 JSON | 后端客户端区域提供唯一“导出总 JSON”入口，新增 shared Web 服务默认复用已有共享后端客户端，同一台后端机器的多个服务会合并导出，避免逐服务导入覆盖客户端配置 |
| TUN 共存 | 本地客户端会检测 Shadowrocket、Clash 等 TUN/fake-ip 环境，bootstrap、启动或重新导入配置后尽量给 Bridge reverse 出站写入物理网卡 `sockopt.interface`，控制台提供“应用代理兼容”一键修复 |
| 自动应用 | 面板保存、启用、停用或删除内网穿透映射后，会立即刷新 VPS 侧 Xray portal 和 Nginx HTTPS；如果应用失败会回滚 catalog，避免客户端已有服务但公网入口仍未生效 |
| 通用客户端资产 | 同一 release 提供 macOS zip、Linux tar.gz、Windows zip，包含安全的 profile 示例和 bootstrap helper |
| 兼容原模式 | launchd / systemd / Scheduled Task 后台运行方式不变，不影响 VPS 侧 portal、Nginx HTTPS 或普通代理节点 |

## 使用方式

1. 管理员先把客户子域名 A/AAAA 解析到 VPS。面板会在“公网域名”输入框里提示候选/已有域名中可用的域名，并隐藏面板域名、普通代理节点域名和未解析到 VPS 的域名。
2. 管理员在“内网穿透”页创建服务映射。公开 HTTPS 可选择提示域名或输入新域名，保存时会校验解析；私有 TCP/SSH 不需要域名。多个 Web 服务默认复用已有 shared Bridge ID；SSH 只保留一个全局默认映射，需要隔离时再手动新增独立 TCP。
3. 在“后端客户端”区域点击“下载配对安装包”，或点击“导出总 JSON”进行手动导入。同一个 shared Bridge ID 只需要下载或导出一次；多个服务会合并到同一个客户端配置。
4. 客户解压包并运行 `install-macos.sh`、`install-linux.sh` 或 `install-windows.ps1`。
5. 安装脚本先运行 `bootstrap-agent.py`，写入本地 `xray-bridge.json`、`bridge-dashboard.json`、`agent-state.json`，然后继续安装 Xray 后台服务。
6. 客户运行 `open-dashboard`，访问 `http://127.0.0.1:19090/` 查看状态；服务映射的编辑、启停和删除仍在面板“服务映射”区域完成。

如果客户电脑同时开启 Shadowrocket、Clash 等本地代理，本地控制台会在“本地代理兼容”里显示系统 DNS/TUN、VPS 路由网卡和 Bridge 已配置网卡。出现公网回环测试异常时，先点击“应用代理兼容”，再重启 Bridge；这只影响 Bridge 到 VPS 的 reverse 出站，不会修改客户的系统代理或普通代理节点。

手动 JSON 导出仍然保留，用于离线环境、客户机器无法访问面板或排障场景。当前版本不包含面板远程执行命令、远程日志流或 Agent 自动更新；这些能力只保留字段和接口命名空间，避免未来扩展时破坏兼容性。

## 升级说明

无需数据库 schema 迁移。已有 v3.0.0 穿透节点继续可用；重新从面板生成配对 Agent 包即可获得 bootstrap 和新版本地控制台，原静态后端安装包和 JSON 导出仍可继续使用。
