# fake-ui v3.1.0

v3.1.0 把“本地服务发布”和“远程访问”合并进同一套本地客户端配对流程。一个本地客户端一次配对后，可以同时承载 HTTPS/TCP/SSH 端口转发，以及可选的 Hysteria2 UDP 443 + WireGuard 远程访问能力。它不绑定任何固定客户域名、设备名称或 VPS IP；新 VPS 安装完成后，管理员在面板里添加服务或设备即可生成对应配置。

## 关键变化

- 新增管理页：`远程访问`
- 新增 API：`/api/desktops`、`/api/desktops/save`、`/api/desktops/action`、`/api/desktops/apply`
- 新增下载：每台设备可下载 `*-remote-desktop-agent.zip`
- 新增导出：每台设备可导出 `*-wireguard.conf`
- 新增 Hysteria2 合并逻辑：把启用设备的 `desktop-*` 账号加入 `auth.userpass`
- 新增统一 Agent bootstrap：`/api/agents/bootstrap` 可返回 `tcp_tunnel`、`remote_desktop`、`proxy_compat` 能力
- 新增代理兼容模板：为 Shadowrocket、Clash/Mihomo、Surge、sing-box、v2rayN/NekoRay 生成 DIRECT 规则建议

## 网络模型

```text
TCP 443
  -> Nginx stream
    -> 面板 HTTPS
    -> VLESS Reality
    -> 本地服务发布 HTTPS

UDP 443
  -> Hysteria2
    -> WireGuard UDP forwarding
      -> macOS / Linux / Windows 虚拟内网
        -> Sunshine / Moonlight / RDP / VNC
```

这意味着远程访问不会抢占 Nginx 的 TCP 443，也不会覆盖 Xray bridge 本地服务发布配置。开启本地代理/TUN 时，HY2 UDP 443 推荐按域名、VPS IP 或客户端进程走 DIRECT；TCP Reality bridge 仍保留自己的直连提示。

## 使用流程

1. 在面板添加控制端和被控主机，给每台设备分配不重复的 WireGuard IP。
2. 点击“应用远程访问配置”，让 Hysteria2 识别这些 `desktop-*` 设备账号。
3. 每台设备下载自己的客户端包，安装 Hysteria2 和 WireGuard。
4. 启动 Hysteria2 UDP forwarding 后导入 WireGuard 配置。
5. 使用 Moonlight、RDP、VNC 或其他桌面客户端访问对端 WireGuard IP。

## 兼容性

- 兼容 macOS、Linux、Windows。
- 兼容现有本地服务发布页面。
- 兼容普通 VLESS/Hysteria2 代理节点。
- 兼容原生 Nginx 复用 TCP 443 部署模式。

## 升级说明

从 v3.0.x 升级无需迁移旧数据。远程访问设备保存在 SQLite settings 的 `desktop_catalog` 中，默认为空；只有管理员添加设备并点击应用后，才会重启 Hysteria2。
