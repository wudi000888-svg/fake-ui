# fake-ui v2.0.0

v2.0.0 是 fake-ui 从 v1 功能型面板走向商业化运营底座的一次大版本升级，重点是移动端体验、SQLite 数据层、链上支付、用户/节点精细化管理，以及可重复部署能力。

## 主要变化

| 模块 | 更新 |
|---|---|
| 前端体验 | 重构为移动优先的模块化 SPA，修复侧栏、套餐、订单、付款、设置和管理按钮在移动端挤压或不可点击的问题 |
| 版本展示 | 面板侧栏和登录态 API 显示当前版本号，默认 `v2.0.0` |
| 数据层 | 默认启用 SQLite：`FAKE_UI_STORE=sqlite`、`FAKE_UI_DB=/data/panel/fake-ui.db`，保留 JSON 迁移和导出能力 |
| 缓存 | 增加统一 TTL cache，并提供缓存状态和管理员清理接口 |
| 支付 | 支持 USD 计价的 USDT、USDC、ETH、BNB、BTC 链上收款，管理员只需配置收款地址 |
| 验账 | 支持公共 RPC / BTC 公共 API 冗余查询、自动扫账、TXID 兜底、到账后自动开通或续费 |
| 订单体验 | 区分待付款、已完成、已取消等状态，用户可取消未付款订单，避免历史订单和待处理订单挤在一起 |
| 用户管理 | 管理员可修改用户套餐、流量、到期时间、状态、备注，并可精确控制用户可用节点 |
| Hysteria2 管理 | 独立管理页支持 HTTP/SOCKS5 出口配置、恢复直连、状态回填和移动端展示 |
| 节点同步 | VLESS/H2 保存或刷新后自动回写出口 IP、国家和节点名称，前端立即显示最新节点列表 |
| 备份迁移 | 管理端提供备份、导出和导入能力；安装脚本会运行 JSON 到 SQLite 迁移 |
| 部署脚本 | 更新 fresh install 和 Windows compose 部署脚本，默认按 v2 数据层启动并校验新前端入口 |

## 部署说明

全新 VPS 仍可直接运行：

```bash
sudo bash scripts/install-fresh-vps.sh
```

已有 v1/v1.2 数据升级时，先备份 `data/`，再执行迁移：

```bash
python3 scripts/migrate-json-to-sqlite.py --data-dir data/panel --db data/panel/fake-ui.db
```

生产环境不要把 `.env`、`data/`、证书私钥、真实 RPC API Key、真实收款地址或任何钱包私钥提交到仓库。
