# fake-ui v2.1.2

v2.1.2 是 v2.1.1 安全加固后的缺陷修复版本，重点修复管理面鉴权、API 登录防护一致性、注册密码存储、账号找回和备份恢复安全边界。此版本不包含数据库 schema 迁移。

## 主要修复

| 类型 | 内容 |
| --- | --- |
| 管理员 QR | `/qr/` 管理员节点二维码现在要求 admin session，未登录或普通用户不能生成管理员节点链接 |
| API 登录 | `/api/login` 和页面登录共用登录限速、失败记录、成功清理和审计事件 |
| 注册密码 | 注册申请保存 PBKDF2 hash，不再把申请密码明文写入 SQLite 或备份 |
| 自助注册 | 管理员可在设置页开启公开注册；用户自助注册成功后返回登录页，默认无套餐、无节点权限，需自行购买套餐 |
| 登录注册 | 登录页与注册页拆分；注册关闭时登录页不显示注册入口，开启时显示注册按钮 |
| 登录反馈 | 修复旧登录 cookie 影响公开注册、注册成功后不跳转、登录后短暂空白和错误提示不可见的问题 |
| 找回密码 | 管理员可开启邮箱验证码找回密码，SMTP 服务商配置单独保存，SMTP 密码不会在 API 响应中回显 |
| 用户资料 | 用户中心支持补充邮箱，管理员和用户 shell 均提供退出登录按钮 |
| 用户管理 | 修复 SQLite-only 用户删除只写审计但用户记录仍保留的问题 |
| 套餐显示 | 修复套餐改名后管理员用户列表和用户首页套餐名不同步的问题 |
| 原生 Nginx | fresh install 生成的 native Nginx 配置补齐安全响应头 |
| 备份恢复 | 恢复包增加大小限制，恢复后的 SQLite 数据库会执行 `PRAGMA integrity_check`，数据库相关文件权限收敛为 `0600` |
| 文档 | 架构文档更新到当前 SQLite repository、`security.py`、路由层和响应工具 |

## 升级说明

| 项目 | 说明 |
| --- | --- |
| 支持来源 | v2.1.1 |
| 数据迁移 | 无 schema 迁移 |
| 用户影响 | 公开注册默认关闭；开启后新用户注册成功需回到登录页登录，默认没有套餐和节点权限 |
| 找回密码 | 默认关闭；生产启用前需在管理员设置中配置 SMTP 主机、端口、账号、密码和发件邮箱 |
| 用户管理 | 升级后删除用户会真实删除 SQLite 用户记录；套餐改名后用户列表会实时显示新套餐名 |
| 生产建议 | 升级前备份 `data/`、Xray 配置和 Hysteria2 配置；升级后重新加载原生 Nginx 配置 |
