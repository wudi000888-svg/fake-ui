# fake-ui v2.1.2

v2.1.2 是 v2.1.1 安全加固后的缺陷修复版本，重点修复管理面鉴权、API 登录防护一致性、注册密码存储和备份恢复安全边界。此版本不包含数据库 schema 迁移。

## 主要修复

| 类型 | 内容 |
| --- | --- |
| 管理员 QR | `/qr/` 管理员节点二维码现在要求 admin session，未登录或普通用户不能生成管理员节点链接 |
| API 登录 | `/api/login` 和页面登录共用登录限速、失败记录、成功清理和审计事件 |
| 注册密码 | 注册申请保存 PBKDF2 hash，不再把申请密码明文写入 SQLite 或备份 |
| 原生 Nginx | fresh install 生成的 native Nginx 配置补齐安全响应头 |
| 备份恢复 | 恢复包增加大小限制，恢复后的 SQLite 数据库会执行 `PRAGMA integrity_check`，数据库相关文件权限收敛为 `0600` |
| 文档 | 架构文档更新到当前 SQLite repository、`security.py`、路由层和响应工具 |

## 升级说明

| 项目 | 说明 |
| --- | --- |
| 支持来源 | v2.1.1 |
| 数据迁移 | 无 schema 迁移 |
| 用户影响 | 已有注册申请如果来自旧版本，仍可按旧数据审批；新注册申请不再保存明文密码 |
| 生产建议 | 升级前备份 `data/`、Xray 配置和 Hysteria2 配置；升级后重新加载原生 Nginx 配置 |
